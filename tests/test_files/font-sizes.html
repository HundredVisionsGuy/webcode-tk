Font-size Experiment

An experiment for testing computed font sizes.

General Notes (font-size of initial)

  • The base font-size by default is 16px
  • I set the body element to 20px to make font-size calculations a little simpler

Keywords

Absolute Keywords

Absolute keywords are always set in reference to the root font size. Each size in the list below will have that size applied.

  • xx-small
  • x-small
  • small
  • medium
  • large
  • x-large
  • xx-large
  • xxx-large

Relative Size Keywords

There are just two relative size keywords: smaller and larger. These bump up or down based on the computed font-size

The keyword larger makes the text 120% larger than whatever the computed font would be.

The keyword smaller makes the text 83.3% smaller than the computed font. Below are some examples.

A div of spans within spans within spans
Using all spans of 110%, we have a div of spans within spans within spans - but this span should be 16px because it has a class of medium thereby proving that when using percentages (just like EMs), they are based on containing element's computed size.

A Note About headings <h1>–<h6>

As you may recall, this page was set to a base font-size of 20px from the default size of 16px. This means, every font is increased by 125%, so an <h2> tag, which is 24px by default, is increased to 30px in this page (just check the inspector).

Wait! Why are the headers below smaller?

Ah, so you checked the inspector did you? Well, the article (the containing element) was set to a base font size of medium, so all headers (except the <h1> element are at their standard size.

<h2>: Heading Level 2 (default size of 24px - 1.5rem)

<h3>: Heading Level 3 (default size of 18.72px - 1.17rem)

<h3 class="larger">: Heading Level 3 with a larger font-size applied inside a 16px parent (result: 19.2px)

<h4>: Heading Level 4 (default size of 16px - 1rem)

<h5>: Heading Level 5 (default size of 13.28px - 0.83rem)
<h6>: Heading Level 6 (default size of 10.72px - 0.67rem)

Notes

  • Why the <h1> tag is smaller in the context of an <article> tag is not readily apparent.
  • All sizes listed above are based on a Chrome browser at 100% viewing size.
